인터페이스 = 규격
다중 상속

자바의 인터페이스
- 클래스가 구현해야 할 메서드들이 선언되는 추상형
- 인터페이스 선언
  - interface 키워드로 선언 ex) public interface SerialDriver{...}

자바 인터페이스에 대한 변화
- Java 7까지
  - 인터페이스는 상수와 추상 메서드로만 구성
- Java 8부터
  - 상수와 추상메서드 포함
  - default 메서드 포함 (Java 8)
  - private 메서드 포함 (Java 9)
  - static 메서드 포함 (Java 9)

여전히 인터페이스에는 필드(멤버 변수) 선언 불가

자바 인터페이스의 전체적인 특징
- 인터페이스는 객체 생성이 불가하다
- 인터페이스 타입의 레퍼런스 변수 선언 가능
- 인터페이스 구현(상속 받은 것처럼 사용하는 것)
   - 인터페이스를 상속 받는 클래스는 인터페이스의 모든 추상 메서드 반드시 구현
- 다른 인터페이스 상속 가능
- 인터페이스의 다중 상속 가능
인터페이스는 인터페이스끼리 상속 가능
클래스와 인터페이스 사이에는 상속이 불가능하지만 구현은 가능하다(inmplements)
