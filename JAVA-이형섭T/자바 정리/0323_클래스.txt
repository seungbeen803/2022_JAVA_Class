메서드 오버로딩(겉은 다르고 속은 같다) : 같은 이름의 메서드를 매개변수의 타입, 
개수를 다르게 하여 여러 개를 생성하는 것, 리턴 타입도 다르다
이름만 같고, 리턴 타입, 매개변수 타입 및 개수, 데이터 타입이 다 달라야한다.

- 같은 이름의 메서드나 생성자를 매개변수의 
  개수나 타입이 서로 다른 여러 개를 만드는 것

new키워드를 이용하여 객체 생성
new : 메모리에 객체 할당

레퍼런스 : 변수를 간접적으로 가리킴 (클래스 타입, 사용자 정의형 타입)
 - 객체 연결을 위한 하나의 타입 장치

객체 소멸 
- new에 의해 할당된 객체 메모리를 자바 가상 기계(JVM)의 가용 메모리로 되돌려 주는 행위

가비지
- 사용이 끝난 객체

가비지 컬렉션
   - 자바에서 가비지를 자동(JVM) 회수하는 과정
   - 가비지 컬렉션 스레드에 의해 수행
- 개발자에 의한 강제 가비지 컬렉션

자바는 패키지 기반이다
하나 이상의 패키지, 클래스를 만들어주어야 자바 응용프로그램이 실행 됨

패키지 : import 등

접근 지정자
 - 자바의 접근 지정자
- private : 외부로부터 완벽차단(동일 클래스 내에서만 허용, getter, setter이용)
- protected : 한 곳에 넣어 보관하겠다(동일 패키지와 자식 클래스에 허용)
- public : 모든 클래스에 허용
- 디폴트(접근지정자 생략) : 소장용의 개념 새 것 뜯지 않음(동일 패키지에 허용)


static - 공유하는 것 
- 클래스당 하나만 생성됨 (클래스 멤버라고도 부름)

final - 변경 불가
